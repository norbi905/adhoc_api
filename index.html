Orbit Mode
Camera orbits building
›

The camera rotates around the building exterior. User can adjust orbit speed, angle, and zoom. Default mode on application load.

Activate
EMIT camera.setMode
{ "action": "camera.setMode", "mode": "orbit", "targetActor": "Building_01", "radius": 5000, "height": 1200, "speed": 0.3 }
Controls
EMIT orbit.setParams
{ "action": "orbit.setParams", "speed": 0.5, "radius": 4000, "height": 800, "autoRotate": true }
Response
LISTEN camera.modeChanged
UE5 confirms orbit mode active
{ "event": "camera.modeChanged", "mode": "orbit", "status": "active" }
↔ Can transition to: Unit View, Walking, POI
🏢
Unit View Mode
Browse & filter units
›

Displays all condo units as interactive 3D boxes on the building. Supports filtering by availability, price range, and number of bedrooms. Works together with Floor Cutoff mode.

Activate
EMIT camera.setMode
{ "action": "camera.setMode", "mode": "unitView" }
Filter Units
EMIT units.applyFilter
{ "action": "units.applyFilter", "availability": "available", "priceMin": 400000, "priceMax": 900000, "bedrooms": [1, 2], "floors": [3, 4, 5, 6] }
Select Unit
EMIT units.selectUnit
{ "action": "units.selectUnit", "unitId": "U-0304", "highlight": true }
EVENT units.unitClicked
User clicks a unit in UE5 viewport — UE5 notifies Vue
{ "event": "units.unitClicked", "unitId": "U-0304", "floor": 3, "bedrooms": 2, "status": "available" }
Clear Filter
EMIT units.clearFilter
{ "action": "units.clearFilter" }
↔ Can activate Floor Cutoff on unit select
🚶
Walking Mode
Street-level exploration
›

First-person street-level navigation around the building exterior. Equivalent to Google Street View. The user controls movement via arrow keys or on-screen controls.

Activate
EMIT camera.setMode
{ "action": "camera.setMode", "mode": "walk", "spawnPoint": "entrance_north", "eyeHeight": 170 }
Teleport to Point
EMIT walk.teleport
{ "action": "walk.teleport", "spawnPoint": "lobby", "facingYaw": 180.0 }
Input Passthrough
EMIT walk.move
For virtual joystick/button UI controls
{ "action": "walk.move", "direction": "forward", "speed": 1.0 }
Response
EVENT walk.positionUpdate
UE5 streams position for minimap sync
{ "event": "walk.positionUpdate", "x": 1240.5, "y": 330.0, "yaw": 92.3 }
↔ Can transition to: Orbit, POI
🗺️
POI Mode
Aerial points of interest
›

Camera ascends above the clouds and looks straight down — a Google Maps-style aerial view. Shows nearby points of interest: schools, transit, shops, parks annotated around the building.

Activate
EMIT camera.setMode
{ "action": "camera.setMode", "mode": "poi", "altitude": 80000, "transitionDuration": 2.5 }
Toggle POI Categories
EMIT poi.setCategories
{ "action": "poi.setCategories", "visible": [ "transit", "schools", "parks", "shopping", "restaurants" ] }
Highlight POI
EMIT poi.highlight
{ "action": "poi.highlight", "poiId": "subway_king_st", "zoomTo": true }
POI Click Event
EVENT poi.markerClicked
User clicks a POI marker in the viewport
{ "event": "poi.markerClicked", "poiId": "subway_king_st", "category": "transit", "name": "King St Station", "distanceM": 320 }
↔ Can transition to: Orbit, Walking
✂️
Floor Cutoff Mode
Sub-mode of Unit View — removes upper floors to reveal interior
›

Activated when a user selects a specific unit in Unit View Mode. The building geometry above the selected floor is clipped/hidden, exposing the floor plan and interior space of the unit. Deactivated by exiting unit selection.

Activate (on unit select)
EMIT building.setFloorCutoff
{ "action": "building.setFloorCutoff", "enabled": true, "floor": 3, "unitId": "U-0304", "animateCutoff": true, "cutoffDuration": 1.2 }
Adjust cutoff level
EMIT building.setCutoffFloor
{ "action": "building.setCutoffFloor", "floor": 5 }
Deactivate
EMIT building.setFloorCutoff
{ "action": "building.setFloorCutoff", "enabled": false }
Response
LISTEN building.cutoffReady
UE5 confirms cutoff animation complete
{ "event": "building.cutoffReady", "floor": 3, "unitId": "U-0304", "status": "visible" }
⚠️ Sub-mode: only active inside Unit View Mode. Exits when unitId is deselected or mode changes.

// Mode Transition Matrix

From To Trigger API Call Required Notes
INIT → ORBIT App loads app.init → auto-transition Default mode on startup
ORBIT → UNIT VIEW User clicks "View Units" button camera.setMode { mode: "unitView" } Camera adjusts to building overview angle
ORBIT → WALKING User clicks "Street View" button camera.setMode { mode: "walk" } Spawns at default entrance point
ORBIT → POI User clicks "Area Map" button camera.setMode { mode: "poi" } Camera ascends with cinematic transition
UNIT VIEW → FLOOR CUTOFF User selects a unit building.setFloorCutoff { enabled: true, floor, unitId } Sub-mode; Unit View stays active
FLOOR CUTOFF → UNIT VIEW User deselects unit or clicks back building.setFloorCutoff { enabled: false } Building geometry restores
UNIT VIEW → ORBIT User clicks "Back" / mode toggle camera.setMode { mode: "orbit" } Resets filters; disables floor cutoff
WALKING → POI User clicks "View Area Map" camera.setMode { mode: "poi" } Camera lifts from street to aerial
WALKING → ORBIT User clicks "Orbit View" camera.setMode { mode: "orbit" }
POI → Any User clicks a mode button camera.setMode { mode: "..." } Camera descends back to scene level

// Global / Cross-Mode API Calls

EMIT app.setLanguage
Change UI language across all modes
{ "action": "app.setLanguage", "locale": "fr-CA" }
EMIT app.setQuality
Adjust render quality / streaming bitrate hint
{ "action": "app.setQuality", "preset": "high" }
EMIT app.setTimeOfDay
Adjust sun/sky lighting across all modes
{ "action": "app.setTimeOfDay", "hour": 14.5 }
EVENT app.error
UE5 reports a runtime error to Vue
{ "event": "app.error", "code": "UNIT_NOT_FOUND", "unitId": "U-9999", "message": "Unit does not exist" }
EMIT app.reset
Hard reset — return to INIT state
{ "action": "app.reset" }
LISTEN camera.modeChanged
Fired after every successful mode change
{ "event": "camera.modeChanged", "mode": "walk", "previousMode": "orbit", "status": "active" }
